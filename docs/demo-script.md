# Demo Script

## 20-Second Setup

“TRON PayFlow is a merchant checkout product on TRON Nile. A merchant issues an invoice in USDT, the payer can settle with supported TRC-20 assets, and the app maps the transaction back to a verified payment receipt.”

## 45-Second Walkthrough

1. Open the merchant dashboard.
2. Point out:
   - merchant wallet
   - support email
   - invoice expiry
3. Create a new invoice.
4. Open the payer link.
5. Connect TronLink and compute the route.
6. Call out:
   - total input value
   - fees
   - slippage buffer
7. Execute the payment.
8. Show:
   - tx hash
   - TronScan link
   - on-chain request id
   - reconciliation hash

## 20-Second Why It Matters

“This is more than a token transfer. It gives merchants real invoicing, payer-friendly routing, failure recovery, support metadata, and reconciliation. The state shown in the UI maps back to the contract’s recorded request summary on Nile.”

## 15-Second Edge Case Option

If time allows, show a failed flow:

- expired invoice
- insufficient liquidity
- out of energy

Then point out the recovery guidance and the merchant-side attempt history.

## Final Judge Close

“We focused on merchant acquiring and checkout because adoption depends on the full product journey: setup, payment, proof, and support. TRON PayFlow turns DeFi routing into a merchant experience a real business could understand and use.”
